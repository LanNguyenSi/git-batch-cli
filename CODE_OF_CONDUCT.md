# Code of Conduct

## Our Standard

Participants are expected to be respectful, constructive, and professional in all project spaces.

Examples of expected behavior:

- giving actionable technical feedback
- assuming good intent
- being respectful in disagreement
- focusing criticism on ideas, code, and behavior

Examples of unacceptable behavior:

- harassment or intimidation
- personal attacks
- discriminatory language or conduct
- doxxing or publishing private information without consent

## Enforcement

Project maintainers may remove or reject contributions, comments, or participation that violate this code of conduct.

## Scope

This code of conduct applies to issue trackers, pull requests, discussions, and other project communication channels.
